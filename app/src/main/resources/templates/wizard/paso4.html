Sistema CI/CD – Paso 4

Paso 4 – Configurar variables AWS + comprobar imagen en ECR

Ver guía de este paso

En este paso introduces las variables necesarias para acceder a AWS/ECR y se comprueba que existe una imagen Docker en Amazon ECR con el repositorio y tag indicados. Si todo sale OK, al final verás las variables para copiarlas como Secrets.

Variables para crear como Secrets
Nombre del repositorio en ECR (sin URL).
Región donde está tu ECR (por ejemplo: eu-west-1).
ID de cuenta AWS (12 dígitos).
Se guarda como secret. Se muestra oculto (••••••). Pulsa Copiar para copiarlo.
Comprobación de imagen
Tag de la imagen en ECR (por ejemplo: latest, v1.0.0).

Estado: PENDIENTE

Ahora crea en tu repositorio estas variables secretas:
Variable Valor Acción
ECR_REPOSITORY
AWS_REGION

AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY ••••••••••••••••
Consejo: en GitHub → Settings → Secrets and variables → Actions → New repository secret.
en GitLab → Settings → CI/CD → Variables → Add variable.
en Jenkins → Manage Jenkins → Credentials → (Global / Folder) → Add Credentials.
